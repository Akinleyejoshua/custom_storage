Authentication Required

To list, upload, or delete assets, your HTTP client must send one of the following authentication headers.

# Option A (Standard Bearer Auth - Raw or Base64 encoded email)

Authorization: Bearer your-email@domain.com

# Option B (Custom Workspace Header)

X-User-Email: your-email@domain.com

POST

/api/auth/login

Validate Workspace Session

Registers or verifies an active workspace session for a user email address.

Request Payload (JSON)

{
  "email": "user@example.com"
}

Success Response (200)

{
  "success": true,
  "message": "Login successful",
  "email": "user@example.com"
}
POST

/api/assets

Upload Documents & Media

Upload images, videos, documents (PDF, Excel, Word, Text), PowerPoint, audio, or archive files to the gateway. Supports multipart/form-data encoding.

Form Parameters

Field Type Required Description
file File Binary Yes The file to upload (Max 500MB)

Success Response (200)

{
  "success": true,
  "data": {
    "assetId": "48b17b2bca6937bc38abda8c9735d1f1",
    "originalFilename": "invoice_Q2.xlsx",
    "secureFilename": "1625123456-e91b5c46.xlsx",
    "mimeType": "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
    "fileSize": 1048576,
    "storagePath": "1625123456-e91b5c46.xlsx",
    "publicUrl": "http://localhost/uploads/1625123456-e91b5c46.xlsx",
    "user_email": "user@example.com",
    "createdAt": "2026-06-19 18:30:00"
  }
}
GET

/api/assets

List Personal Files

Retrieve a paginated list of all uploaded assets belonging to the authenticated workspace email, sorted by creation date (newest first).

Query Parameters

Parameter Type Default Description
page Integer 1 Page number to view
limit Integer 20 Number of results per page

Success Response (200)

{
  "success": true,
  "data": [
    {
      "assetId": "48b17b2bca6937bc38abda8c9735d1f1",
      "originalFilename": "invoice_Q2.xlsx",
      "mimeType": "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
      "fileSize": 1048576,
      "publicUrl": "http://localhost/uploads/1625123456-e91b5c46.xlsx",
      "user_email": "user@example.com",
      "createdAt": "2026-06-19 18:30:00"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 1,
    "pages": 1
  }
}

DELETE

/api/assets/{assetId}

Permanently Delete File

Permanently delete a file from both the storage directory and the database index. Only permitted if the authenticated user owns the asset.

Success Response (200)

{
  "success": true,
  "message": "Asset deleted successfully",
  "assetId": "48b17b2bca6937bc38abda8c9735d1f1"
}

POST

/api/folders

Create Folder

Creates a new virtual folder for organizing your workspace assets.

Request Payload (JSON)

{
  "name": "Design Assets"
}

Success Response (200)

{
  "success": true,
  "message": "Folder created successfully",
  "folderId": "5f1a3b8c-1e2a-4bcd-86ef-d78a9c2e0b3c",
  "name": "Design Assets"
}
DELETE

/api/folders/{folderId}

Delete Folder

Permanently delete a folder. All files contained inside the folder will automatically move back to the Root directory (none are deleted from storage or database).

Success Response (200)

{
  "success": true,
  "message": "Folder deleted successfully. Assets moved to root.",
  "folderId": "5f1a3b8c-1e2a-4bcd-86ef-d78a9c2e0b3c"
}
POST

/api/assets/{assetId}/move

Move File to Folder

Move an asset to a virtual folder. This only modifies the file's folder metadata in the database; the file's disk storage path and public URL remain unchanged.

Request Payload (JSON)

{
  "folderId": "5f1a3b8c-1e2a-4bcd-86ef-d78a9c2e0b3c"  // Use null to move file back to root
}

Success Response (200)

{
  "success": true,
  "message": "Asset moved successfully",
  "assetId": "48b17b2bca6937bc38abda8c9735d1f1",
  "folderId": "5f1a3b8c-1e2a-4bcd-86ef-d78a9c2e0b3c"
}
POST

/api/assets/move

Batch Move Files

Move multiple assets in a single batch request to a virtual folder or root level.

Request Payload (JSON)

{
  "assetIds": ["id1", "id2"],
  "folderId": "5f1a3b8c-1e2a-4bcd-86ef-d78a9c2e0b3c"  // Or null for Root
}

Success Response (200)

{
  "success": true,
  "message": "2 assets moved successfully",
  "folderId": "5f1a3b8c-1e2a-4bcd-86ef-d78a9c2e0b3c"
}
DELETE

/api/assets

Batch Delete Files

Permanently delete multiple assets from storage and database indexing in a single request.

Request Payload (JSON)

{
  "assetIds": ["id1", "id2"]
}

Success Response (200)

{
  "success": true,
  "message": "2 assets deleted successfully",
  "assetIds": ["id1", "id2"]
}
POST

/api/assets/{assetId}/rename

Rename File Metadata

Rename the original filename attribute of an asset in the database. This does not change the physical storage path or the public URL.

Request Payload (JSON)

{
  "filename": "new_invoice_Q2.xlsx"
}

Success Response (200)

{
  "success": true,
  "message": "Asset renamed successfully",
  "assetId": "48b17b2bca6937bc38abda8c9735d1f1",
  "filename": "new_invoice_Q2.xlsx"
}

Terminal Quick Start

Upload a secure document from your terminal using cURL:

curl -X POST \
  -H "Authorization: Bearer user@example.com" \
  -F "file=@/path/to/invoice.xlsx" \
  http://localhost/api/assets
